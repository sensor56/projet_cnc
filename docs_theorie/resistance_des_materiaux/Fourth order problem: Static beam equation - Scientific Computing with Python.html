Fourth order problem: Static beam equation

10/3/2013

1 Comment

 
The Euler-Bernoulli equation describes the deflection of a beam due to an applied load. If the elastic modulus of the material and the second moment of inertia are uniform, and the beam is ideally thin, we  can model this with the simple fourth-order differential equation.
d4udx4=f(x)
In the equation f(x) is a distributed load and u(x) is the amount of deflection. Since the problem is fourth order, we can, in general, prescribe boundary conditions involving u and its first three derivatives at the end points. If the beam is clamped at x=0 and subjected to a point force a at x=1, the boundary conditions are
u(0)=0,u′(0)=0,u′′(1)=0,u′′′(1)=a

In the case where a=0, the right end of the beam is free.  Since the finite difference method typically has one equation per grid point, sometimes people get confused as to how we can specify two conditions at one point. 

To numerically solve this problem with finite differences we need the difference approximations to the first four derivatives (to second order). Using the code ufdwt.py to compute centered difference approximations, we get

The difference approximations are then
u′ku′′ku′′′ku′′′′k≈12h[uk+1−uk−1]≈1h2[uk+1−2uk+uk−1]≈12h3[uk+2−2uk+1+2uk−1−uk−2]≈1h4[uk+2−4uk+1+6uk−4uk−1+uk−2]

The fourth derivative approximation above works fine as is for k=2,⋯,n−2, however, special care is needed when the boundary points are involved. Since u0=0 is known, we do not need to write an equation for u0, however, there is an equation for u1:
u3−4u2+6u1−4u0+u−1=h4f1

Since x−1 is outside of the domain, we have added an additional degree of freedom to the problem. To accomodate this, we need an extra equation, in particular the condition u′0=0, which gives us
u1−u−1=0

At the other end, we have the equation
un+2−4un+1+6un−4un−1+un−2=h4fn

To eliminate the extra degrees of freedom, we use the boundary conditions 
u′′n=0 and u′′′n=a. The finite difference approximation to these equations can be written as
(0−112−201−201)⎛⎝⎜⎜⎜⎜⎜⎜⎜un−2un−1unun+1un+2⎞⎠⎟⎟⎟⎟⎟⎟⎟=⎛⎝⎜02h2ah4fn⎞⎠⎟

Solving for exterior values in terms of interior values
(un+1un+2)=(02h3a)−(0−114−2−4)⎛⎝⎜un−2un−1un⎞⎠⎟

Using these in in the un−1 equation, we obtain
un−3−4un−2+5un−1−4un=h4fn


and in the un equation:
2un−2−4un−1+2un=h4fn−2h3a


Actually, we will want to divide this equation by 2 to make the matrix symmetric
⎛⎝⎜⎜⎜⎜⎜⎜⎜⎜⎜⎜⎜⎜⎜⎜⎜⎜⎜7−41−46−411−46−41⋱⋱⋱⋱⋱1−46−411−45−21−21⎞⎠⎟⎟⎟⎟⎟⎟⎟⎟⎟⎟⎟⎟⎟⎟⎟⎟⎟⎛⎝⎜⎜⎜⎜⎜⎜⎜⎜⎜⎜⎜⎜u1u2u3⋮un−2un−1un⎞⎠⎟⎟⎟⎟⎟⎟⎟⎟⎟⎟⎟⎟⎛⎝⎜⎜⎜⎜⎜⎜⎜⎜⎜⎜⎜⎜h4f1h4f2h4f3⋮h4fn−2h4fn−112h4fn−h3a⎞⎠⎟⎟⎟⎟⎟⎟⎟⎟⎟⎟⎟⎟


The Python implementation of the method follows 
import numpy as np
from scipy.linalg import solveh_banded
import matplotlib.pyplot as plt

def beam4(n,ffun,a):
    """
    Solve the equation u""=f(x) on [0,1] with the boundary conditions
    u(0) = u'(0) = u"(1) = 0,  u"'(1) = a
    using centered differences
    """
    
    x = np.linspace(0,1,n+1)
    h = 1.0/n
    
    # Allocate bands of the symmetric matrix
    stencil = np.array((1,-4,6))
    B = np.outer(stencil,np.ones(n))
    B[1,-1] = -2;    B[2,0] = 7;    B[2,-1] = 1;    B[2,-2] = 5    
    
    f = ffun(x)
    f *= h**4;    f[-1] *= 0.5;    f[-1] -= a*h**3
    
    u = np.zeros(n+1)    
    u[1:] = solveh_banded(B,f[1:],lower=False)
    
    return x,u
To test the code out, we can choose an f(x) and a where we know the exact solution. In particular if f(x)=−(π2)4cos(π2x) and a=−(π2)3, then the exact solution is u(x)=1−cos(π2x).
if __name__ == '__main__':
    
    n = 100    
    
    a = -(np.pi/2)**3
    
    def ffun(y):
        return  -(np.pi/2)**4*np.cos(np.pi*y/2)

    x,u = beam4(n,ffun,a)        

    # Exact solution    
    uex = 1-np.cos(np.pi*x/2)
    plt.plot(x,u)
    plt.plot(x,uex)
    
    plt.show()
Calculating the pointwise error as a function of h, we'll find that it initially decreases as O(h2), however, there is a limit to how small the error will get. 
def beam4error(n):
    
    a = -(np.pi/2)**3
    
    def ffun(y):
        return  -(np.pi/2)**4*np.cos(np.pi*y/2)

    x,u = beam4(n,ffun,a)        
    uex = 1-np.cos(np.pi*x/2)
    
    return max(np.abs(u-uex)) 
We can compute the error for a range of h with this test script
if __name__ == '__main__':
    
    n = np.round(np.logspace(1,4,100))
    e = np.zeros(100)
    
    for k in xrange(100):
        e[k] = beam4error(n[k])
        
    plt.loglog(1.0/n,e)
    plt.loglog(1.0/n,1.0/n**2,'--')    
    plt.xlabel('h')
    plt.ylabel('error')
    plt.show()

Error of the finite difference solution of the fourth-order problem

 The reason that we can only get about six or seven digits correct is due to the conditioning of the matrix. As a general rule of thumb the condition number of finite difference matrix corresponding to the kth derivative will increase as O(nk), assuming a uniform grid. We can obtain better conditioning by writing the fourth order problem as a 
 system of two second order problems.
