by bernd » Tue Oct 07, 2014 6:14 pm

played a bit with Cantileverbeam too ... and got results as expected. I'm not finished but since I don't know when I will have time again I'm gone post it anyway.

I did not change the MechanicalAnalysis.py I changed the CalculiX input file instead and made a manual calculation of the file and used CGX for the output.

Here is what I did. I took the expample of Calculix Manual beamp.inp A cantilever beam 1 m x 1 m x 8 m loaded by 9 MN at the end. Material is Steel E = 210000 MN / m2

Displacement (Formula see first Post)
f = (F x l^3) / (3 x E x I) = ( 9 x 8^3 ) / ((3 x 210000 x ((1x1^3)/12)) = (12x9x8^3) / (3x210000) = 0.0877 m

In CGX use Dataset DISPR and then Entity D2.

beamp.inp (original CalculiX) has 8.77e-002 = 0.0877 m
Box_Mesh.inp (original FreeCAD) has wrong Material and wrong Loaddirection and wrong load (you can not put in gui 0.009)
Box_Mesh-edited.inp has 8.79e-002 = 0.0879 m

If FreeCAD document is opend the Forces are shown the wrong direction, just click on the Forceconstrained and click ok and they will show to y-Dircetion. At the end this does not matters becaus force is exported the wrong direction either.

Next to do ... If i have a minute I will have a look at MechanicalAnalysis.py to get the right result without editing the input file.

BTW: I don't not what it should be but kwahoo is right the force is NOT N/mm2 the force is N !!!

Checked the material too. If youth modulus in FCmat files is integer it is not accepted by the FEMMaterialGui. Change in Steel.FCMat from 2e+08 to 2.0e+08 and it will be accepeted from the FEM object. If one opens the FEMMaterialGui the Material is set back to PLA all the time . I could not find why. If one puts values in the spinn boxes these values are taken by click on ok . Material Values of the FEMObject could be checked by
Code: Select all
App.ActiveDocument.MechanicalMaterial.Material['FEM_poissonratio']
App.ActiveDocument.MechanicalMaterial.Material['Mechanical_youngsmodulus']


If Report View Option Redirect Python Output is activated they are printed on CalculiX run.

In current MechanicalAnalysis.py of FreeCAD the youth modulus is written in kPa to the CalculiX file and the load is multiplied by 1000 to get the same unit. I would prefere the other way around. That would mean write youth modulus in MPa = N/mm2 = MN/m2 and the force like it was put in by user in N or MN. Attached again a MechanicalAnalysis.py which works with the cantilever and makes it possible to change the material (if you have valid materials files besides PLA available).

It took me some time to fully understand the unit system. So here is a little reminder (for me)
In the {mm,N,s,K} system the mass is a derived dimension. It has the unit (N*s²/mm) which can be experessed as Mg (Megagramm).
m/Mg; p/MPa; E/mJ; P/mW; U/mV; R/mOhm; C/kF; B/kT
